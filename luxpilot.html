LuxPilot

A governed LLM advisory system for microenterprises.

LuxPilot gives very small businesses concrete, actionable marketing advice, and it does so under governance. The system separates deterministic routing from probabilistic generation, so its behaviour stays inspectable, testable, and governable.

Architecture

A three-layer design keeps decision logic separate from language generation. The model writes; it never decides governance.

Layer 1 — Versioned domain knowledge

Domain knowledge is versioned, so every recommendation can be traced back to the exact knowledge state that produced it.

Layer 2 — Deterministic rule engine (LLM-free)

An LLM-free engine decides routing, feasibility, and which campaign families apply. Because it is deterministic, the logic is auditable and reproducible.

Layer 3 — Model-agnostic LLM execution

Generation happens only after Layer 2 has decided what is appropriate, under explicit quality gates and human validation. The model works within contracts, not around them.

Product and research instrument

LuxPilot is built as both a commercial product and a research-grade evaluation instrument, with immutable profile snapshots, audit logs, replayable routing decisions, golden-trace fixtures, and evaluation hooks for feasibility, actionability, explainability, and governance integrity.

What changed from the beta

The beta proved the product flow but was not reproducible enough for research-grade evaluation. The rebuild moves routing out of frontend code, replaces hard-coded rules with versioned rules-as-data, decomposes a monolithic prompt into contracted LLM functions, introduces profile snapshot immutability, and replaces funnel analytics with audit and replay infrastructure.
